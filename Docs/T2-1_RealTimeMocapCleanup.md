# T-2.1 실시간 모캡 클린업 모듈 (Real-time Mocap Cleanup)

> **Phase:** 2 — 3단계 모션 파이프라인  
> **담당 파일:** `Assets/ParablePortfolio/Runtime/Motion/MotionCleanupModule.cs`  
> **파이프라인 위치:** Stage 1 (LateUpdate 최상단, IK 이전)

---

## 1. 문제 정의

MediaPipe 웹캠 기반 모캡은 광학식 모캡과 달리 두 가지 잡음이 상시 발생한다.

| 잡음 종류 | 원인 | 증상 |
|-----------|------|------|
| **고주파 지터** | 조명 변화·카메라 센서 잡음·자세 예측 불안정 | 매 프레임 ±소폭 흔들림 |
| **아웃라이어** | 신체 부위 가림(Occlusion) 시 랜드마크 신뢰도 급락 | 1~수 프레임 값 급등 후 복귀 |

이 두 잡음을 제거하지 않으면 아바타가 심하게 떨리고, 팔·다리가 순간적으로 비정상 위치로 점프한다.

---

## 2. 솔루션 아키텍처

```
[MediaPipe 측정값 z_t]
        │
        ▼
[아웃라이어 Rejection]  ← |z_t - x̂_(t-1)| > threshold 이면 z_t = x̂_(t-1)
        │
        ▼
[저역 통과 필터]  ← EMA 또는 칼만 필터 선택
        │
        ▼
[필터링된 muscle 값 x̂_t]
        │
        ▼
[Stage 3 아바타 보정으로]
```

**처리 시점:** `LateUpdate()` — Animator가 Idle 클립을 평가하여 `bodyPosition`을 확정한 뒤, `GetHumanPose` → Stage 1 → `SetHumanPose` 순서로 실행.

---

## 3. 필터 알고리즘 비교

### 3.1 EMA (지수이동평균)

**수식:**
```
x̂_t = α · z_t + (1 - α) · x̂_(t-1)
```

| 파라미터 | 의미 | MediaPipe 권장 |
|----------|------|----------------|
| `α (smoothingAlpha)` | 0~1, 높을수록 측정값 신뢰 | 0.15 ~ 0.25 |

**특징:**
- 구현 단순, Inspector에서 직관적 조정
- 지연이 **고정**: τ ≈ 1/α - 1 프레임 (α=0.25 → ~3 프레임)
- 마커 가림 후 재등장 시 수렴 속도가 안정 구간과 동일 → 복원 느림

**장/단점:**

| 장점 | 단점 |
|------|------|
| 파라미터 1개, 직관적 | 지연 고정 (빠른 동작에서 에너지 손실) |
| 계산 비용 최소 | 마커 재등장 후 복원 느림 |

---

### 3.2 스칼라 칼만 필터

**모델:** 근사 상수 모델 (`x_k = x_(k-1) + w`, w ~ N(0, Q))

**예측 단계:**
```
x̂_k|k-1 = x̂_(k-1)         (이전 추정값 유지)
P_k|k-1  = P_(k-1) + Q     (불확실성 Q만큼 증가)
```

**업데이트 단계:**
```
K       = P_k|k-1 / (P_k|k-1 + R)       (칼만 게인, 0~1)
x̂_k     = x̂_k|k-1 + K · (z_k - x̂_k|k-1) (측정으로 보정)
P_k     = (1 - K) · P_k|k-1              (불확실성 갱신)
```

| 파라미터 | 의미 | MediaPipe 권장 |
|----------|------|----------------|
| `Q (kalmanQ)` | 프로세스 노이즈 — 시스템이 프레임마다 얼마나 변할 수 있는가 | 0.0005 ~ 0.002 |
| `R (kalmanR)` | 측정 노이즈 — 센서(MediaPipe)가 얼마나 불확실한가 | 0.05 ~ 0.2 |

**칼만 게인 K 동작 원리:**
```
K = P / (P + R)

안정 구간:    P ≪ R → K ≈ 0 → 측정 불신, 강한 필터링
불확실 구간: P ≫ R → K ≈ 1 → 측정 신뢰, 빠른 수렴
```

즉, 마커 가림 직후 P가 누적되어 크진 시점에 재등장하면 K가 1에 가까워져 빠르게 복원한다. EMA에서 고정 지연으로 느리게 복원되는 문제를 해결한다.

**장/단점:**

| 장점 | 단점 |
|------|------|
| 가변 게인 → 마커 재등장 후 빠른 복원 | 파라미터 2개 (Q, R) — 직관성 낮음 |
| 안정 구간에서 강한 필터링 | EMA 대비 계산 비용 약 2배 (여전히 O(n) |
| 불확실성 모델 → 노이즈 특성 반영 | muscle 수(~95)에 독립적 배열 필요 |

## 4. EMA vs 칼만 필터 — 언제 무엇을?

| 상황 | 권장 필터 |
|------|-----------|
| 빠른 프로토타이핑, 파라미터 직관성 중요 | EMA |
| 마커 가림이 잦은 환경 (손이 얼굴 앞 등) | Kalman |
| 고품질 광학식 모캡 (잡음 낮음) | EMA (alpha 0.4~0.6) |
| MediaPipe 웹캠 모캡 (잡음 높음) | Kalman (Q=0.001, R=0.08) |
| 포트폴리오 데모 | Kalman (차별점) |

---

## 5. 핵심 트레이드오프 (패러블 아티클)

> *"필터를 과하게 걸면 에너지 손실, 부족하면 지터"*

필터 강도를 높이면(α 낮춤 / R 높임):
- 지터는 감소 ✓
- 퍼포머의 작은 표현 동작이 뭉개짐 ✗ (에너지 손실)

이 트레이드오프는 해결이 불가능한 물리적 한계다. 설정은 공연 유형에 따라 다르게:
- 정밀 표현 공연 → 필터 약하게
- 배경 앰비언트 모션 → 필터 강하게

---

## 6. 아웃라이어 Rejection

```
|z_t - x̂_(t-1)| > threshold  →  z_t = x̂_(t-1)  (last-good 값 유지)
```

**주의:** MediaPipe 웹캠 환경에서는 **threshold = 0 (비활성화)** 권장.  
이유: 팔을 빠르게 뻗는 정상 동작도 muscle delta가 0.3~0.5 이상 나올 수 있어 오탐(False Positive)이 발생한다.  
광학식 모캡처럼 프레임레이트가 높고 잡음이 낮은 환경에서는 0.3~0.5로 설정 유효.

## 7. Inspector 사용 가이드

```
MotionCleanupModule (MotionPipeline 컴포넌트 내)
├── Enabled                   [✓]    ← false = 이 단계 스킵 (A/B 비교용)
├── Filter Mode               [Kalman]
│
├── EMA 파라미터 (EMA 모드일 때)
│   └── Smoothing Alpha       [0.25]
│
├── 칼만 필터 파라미터 (Kalman 모드일 때)
│   ├── Kalman Q              [0.001]   ← 높이면 더 빠르게 추적
│   └── Kalman R              [0.08]    ← 높이면 더 강하게 필터링
│
└── 아웃라이어 제거
    └── Outlier Threshold     [0]       ← MediaPipe는 0 권장
```

**A/B 테스트 방법:**
1. `MotionPipeline.injectNoise = true`, `noiseAmount = 0.15`, `outlierProbability = 0.02`
2. `Enabled = false` → 아바타가 심하게 떨리는 원본 상태 확인
3. `Enabled = true`, `FilterMode = EMA` → EMA 효과 확인
4. `FilterMode = Kalman` → 칼만 효과 비교

## 8. 성능 특성

| 항목 | EMA | Kalman |
|------|-----|--------|
| 시간 복잡도 | O(n) | O(n) |
| 메모리 | float[n] x 1 | float[n] x 2 |
| n (HumanPose.muscles) | ~95개 | ~95개 |
| 60fps 기준 추가 비용 | <0.01ms | <0.02ms |

muscle 수가 약 95개로 고정이므로 두 방식 모두 성능 무시 수준.

## 9. 시행착오 기록

### 9.1 Outlier Threshold 0.2 → 팔이 굳어버리는 문제
**현상:** Outlier Threshold를 0.2로 설정하자 팔 올리기 동작이 아바타에 전혀 반영 안 됨  
**원인:** MediaPipe → HumanPose 변환 결과, 팔을 천천히 올려도 muscle delta가 0.2 이상 나올 수 있음  
**해결:** threshold를 0으로 비활성화. MediaPipe 웹캠 환경에서는 칼만 필터의 가변 게인이 아웃라이어 억제를 어느 정도 흡수.

### 9.2 smoothingAlpha 0.1 → 팔이 반응 안 하는 것처럼 보이는 문제
**현상:** α=0.1 로 낮추자 팔 올리기 동작이 ~9 프레임 지연되어 사실상 반응이 없어 보임  
**원인:** EMA 지연 τ ≈ 1/α - 1 = 9 프레임. 30fps 기준 0.3초 지연.  
**해결:** α=0.25 (τ≈3 프레임)로 조정. 칼만 필터로 전환 시 안정 구간에서는 동등한 스무딩, 빠른 동작에서는 더 빠른 반응.

### 9.3 첫 프레임 초기화 이슈
**현상:** 시작 직후 1프레임 동안 아바타가 이상한 포즈로 점프  
**원인:** _emaState / _kalmanX가 null인 채로 Process() 호출  
**해결:** 첫 프레임은 초기화만 하고 `return` (raw pose 그대로 통과)
